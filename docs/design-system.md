# Design System

IncidentForge uses a restrained product-interface aesthetic for generated reports.
The goal is to make evaluation output feel credible to production engineers while
remaining visually distinctive in a GitHub portfolio.

## Tokens

| Token | Value | Usage |
| --- | --- | --- |
| Background | `#050505` | Main report canvas |
| Text | `#f4f4f0` | Primary foreground |
| Muted | `#a7a7a0` | Secondary labels |
| Fault red | `#ff2a2a` | Root cause and failure signals |
| Panel | `rgba(255,255,255,0.075)` | Transparent report surfaces |
| Line | `rgba(255,255,255,0.18)` | Technical borders and separators |

## Motifs

- Dot matrix marks for identity and signal density
- Transparent panels to expose the report structure
- Monospace labels for machine-readable context
- One red accent for faults, never broad red decoration
- Square corners and restrained borders

## Trademark Note

The design is an original open-source interface inspired by broader monochrome,
transparent, dot-based consumer technology aesthetics. It does not use Nothing
logos, proprietary assets, or product names in generated UI.
